# Custom Creations Unlimited — Premium Custom Branding Website

A fast, accessible, conversion-focused marketing site for **Custom Creations Unlimited**, a custom-branding
house (embroidery, apparel, promo products, awards, laser engraving, personalized gifts).

Hand-built static site — **no build step, no dependencies, no framework**. Just open it.

> **Brand & design system:** see [`BRAND-DESIGN-SYSTEM.md`](./BRAND-DESIGN-SYSTEM.md).

## View it locally

**Option A — just open the file:** double-click `index.html`. Everything works except
the Google Map and web fonts, which need an internet connection (they still load fine).

**Option B — run a local server** (recommended; enables clean relative paths):
```bash
cd Website_New
python3 -m http.server 8000
# then open http://localhost:8000
```
Any static server works (`npx serve`, VS Code Live Server, etc.).

> Note: the in-app Launch preview couldn't run here because this folder sits outside the
> session's project root and the sandboxed preview server can't reach it — the command
> above runs the exact same static server with no restriction.

## What's built (Phases 1–3)

The **complete primary navigation** — **19 pages total**, all cross-linked, all SEO-ready:
a premium **homepage**, **6 service pages**, **Gallery**, **Industries**, **About**,
**Contact**, **Quote Builder**, an **FAQ** page, and a **Blog** (index + 5 articles) —
plus the full **design system** every page reuses.

**FAQ** (`faq.html`) — 20 questions across 6 categories in accessible accordions, with
**FAQPage JSON-LD** so answers can appear directly in Google results.

**Blog** (`blog/`) — generated by [`scripts/build-blog.py`](./scripts/build-blog.py): a
featured post, a filterable article grid, a newsletter CTA, and 5 full articles with a
clean reading layout, author/meta, pull quotes, related posts, and **BlogPosting JSON-LD**.

**Top-level pages** (root) — generated by [`scripts/build-pages.py`](./scripts/build-pages.py):
- **Gallery** — 16-piece filterable portfolio (9 categories) with a keyboard-accessible lightbox.
- **Industries** — 14 detailed industry cards + stats band.
- **About** — story, mission & vision, 6 core values, process, equipment, team, community.
- **Contact** — contact rows, Google Map, and a working contact form.
- **Quote Builder** — multi-section interactive form: service, quantity, need-by date, size &
  color chips, budget, delivery, personalization, **drag-and-drop logo upload**, contact details,
  plus a sticky "how it works" sidebar.

> **Forms are LIVE.** The Contact and Quote forms POST to Formspree
> (`https://formspree.io/f/xykqkqao`) via AJAX, with a spam honeypot + `_subject`. If a request
> ever fails they fall back to composing a pre-filled email, so they degrade gracefully offline.
>
> **First-submission note:** Formspree requires you to confirm the form once — submit either form
> (or check your Formspree dashboard) and click the activation email the first time, after which
> submissions flow straight to your inbox. To change the endpoint later, update the `action` in
> `scripts/build-pages.py` and re-run it.

**Service pages** (`services/`) — each includes: breadcrumb, hero with stat highlights,
overview + capability checklist, benefits grid, popular-products grid, recent-work gallery,
industries served, a testimonial, an FAQ accordion, quote CTA and related-services links.
Every page has its own SEO title/meta, Open Graph, and **Service + BreadcrumbList JSON-LD**.

> **Editing service content:** don't hand-edit the HTML in `services/` — change the data in
> [`scripts/build-services.py`](./scripts/build-services.py) and re-run
> `python3 scripts/build-services.py`. This keeps all six pages consistent.

### Imagery
- **155 photos are placed** site-wide from a library of **67 AI-generated WebP images**
  (`assets/img/`): category shots (embroidery, apparel, promo, awards, engraving, drinkware,
  gifts, branding, studio), 4 team headshots, a hero shot, plus **46 product-specific images**
  (`prod-*.webp`) — one for every card in the service pages' "What we make most" grids.
- [`scripts/apply-images.py`](./scripts/apply-images.py) maps images to slots two ways:
  an exact-title `LABEL_MAP` (used for the product grids, e.g. "Crystal Awards" →
  `prod-crystal.webp`), then category keyword rules for everything else.
- To use your **real product photos**, replace the matching file in `assets/img/` (same name)
  and re-run `python3 scripts/apply-images.py` — or `bash scripts/build-all.sh`.

### Rebuilding the site
Because the page generators emit fresh `.ph` placeholders, **always run `apply-images.py` last**.
The simplest way is the all-in-one script, which runs everything in the right order:
```bash
bash scripts/build-all.sh
```

## Deployment

It's static — host anywhere:

**Netlify / Vercel (drag-and-drop or git):**
1. Push this folder to a git repo (or drag the folder into the Netlify dashboard).
2. No build command. Publish directory = project root.
3. Add your custom domain; HTTPS is automatic.

**GitHub Pages:**
1. Push to a repo, Settings → Pages → deploy from `main` / root.

**Any host / S3 / Cloudflare Pages:** upload the files as-is.
